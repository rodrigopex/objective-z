## Context

Follow-up to OZ-059. The golden tests, test_collect.py, test_resolve.py, and test_e2e.py have been migrated to real `.m` sources compiled through Clang. `test_emit.py` (~4400 lines, ~50+ tests) still constructs synthetic `OZModule` objects with handcrafted `body_ast` dictionaries.

## Input

`test_emit.py` builds `OZModule`/`OZClass`/`OZMethod` objects directly with inline AST JSON for method bodies, then calls `emit()` and checks the generated C output.

## Observed

Tests pass but use simplified AST structures that don't match real Clang output. OZ-059 already proved this hides real bugs (e.g., `nil` init pattern not recognized from real Clang AST).

## Expected

Each test should compile a real `.m` file through Clang → collect → resolve → emit, then assert on the generated C. This is a large rewrite (~50+ tests, each needing equivalent ObjC source and updated assertions for richer Clang output).

## Workaround

None needed — existing tests still validate emit behavior, just with synthetic input.
