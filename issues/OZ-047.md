## Context

Need a Foundation class that stores a non-capturing block and calls it on
dealloc, passing an owner reference. Enables deferred cleanup tied to object
lifecycle — the owner pointer sidesteps the non-capturing block limitation by
serving as the "captured context."

## Input

```objc
_cleanup = [[OZDefer alloc] initWithOwner:self block:^(id owner) {
    /* cleanup using owner */
}];
```

## Observed

No OZDefer class exists yet. Additionally, `__unsafe_unretained` qualifier
on ivars is ignored by the transpiler — it strips the qualifier and treats
the ivar as strong (auto-retained/released), which would cause retain cycles
when OZDefer stores a back-reference to its owner.

## Expected

1. Transpiler respects `__unsafe_unretained` on ivars (see OZ-051)
2. OZDefer Foundation class with `initWithOwner:block:` and `initWithBlock:`
3. Block fires during dealloc with owner reference (or nil)
4. Block-typed ivar generates valid C function pointer struct field syntax (see OZ-049)

## Resolution

Added OZDefer Foundation class with `initWithOwner:block:` and `initWithBlock:`
convenience init. Block fires during dealloc with owner pointer (or nil).
Owner stored as `__unsafe_unretained id` to avoid retain cycles.
Registered in Foundation.h umbrella, transpiler _FOUNDATION_NAMES, and
_OZ_INCLUDE_PREFIXES. PR #21.
