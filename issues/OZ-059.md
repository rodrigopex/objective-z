## Context

~90% of transpiler golden tests use handcrafted JSON ASTs that are simpler than real Clang output. This means tests may pass on synthetic input but miss bugs triggered by real Clang ASTs (extra nodes, different field ordering, richer OZObject from SDK). OZ-058 already proved that real Clang ASTs expose issues synthetic ones hide (generic type erasure).

## Input

16 golden test cases, 15 with only handcrafted `input.ast.json`, 1 with `source.m` (`error_generic_source_mismatch`).

## Observed

Tests validate against synthetic ASTs that don't match real Clang output — false confidence.

## Expected

Each golden test has a `source.m` that Clang compiles to AST JSON. `input.ast.json` is auto-regenerated from `source.m` and kept as fallback for CI without Clang. Two cases (`error_circular_inheritance`, `error_duplicate_method`) must stay handcrafted — Clang rejects/deduplicates them.
