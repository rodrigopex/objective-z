## Context

Discovered while implementing OZ-047 (OZDefer). The transpiler supports
`__unsafe_unretained` on properties (collect.py ownership tracking) but
ignores it on ivars. The qualifier is stripped by `_strip_qualifiers()` before
`is_object` checks, so `__unsafe_unretained id` ivars are auto-retained on
assignment and auto-released in dealloc — identical to strong ivars.

This creates retain cycles when a child object stores a back-reference to its
owner (e.g., delegate pattern, OZDefer owner pointer).

## Input

```objc
@interface Watcher : OZObject {
    __unsafe_unretained id _delegate;
    OZString *_name;
}
@end
```

## Observed

Generated dealloc releases both ivars:

```c
void Watcher_dealloc(struct Watcher *self)
{
    OZObject_release((struct OZObject *)self->_delegate);  /* WRONG */
    OZObject_release((struct OZObject *)self->_name);
    OZObject_dealloc((struct OZObject *)self);
}
```

Ivar assignment also generates retain/release for `_delegate`.

## Expected

`_delegate` must NOT be retained on assignment or released in dealloc:

```c
void Watcher_dealloc(struct Watcher *self)
{
    /* _delegate skipped — __unsafe_unretained */
    OZObject_release((struct OZObject *)self->_name);
    OZObject_dealloc((struct OZObject *)self);
}
```

## Resolution

Added `is_unretained` property to OZType in model.py. Updated 5 locations in
emit.py to filter out `__unsafe_unretained` ivars from auto-retain/release:
`_has_auto_dealloc`, auto-dealloc proto, `_is_object_ivar_assign`,
`_emit_user_dealloc`, and `_emit_auto_dealloc`. Transpiler unit test added.
PR #21.
