# PLAN: `extract.py` + Slab Distribution Per-Class

## Context

Two changes that complement each other:

1. **`extract.py`** — Tree-sitter CST → Jinja2 template string. ObjC constructs become `{{ _n_L_C }}` placeholders with annotating comments about the original code; C code, comments, and whitespace are preserved verbatim. Context values are pre-rendered C from existing AST emitters.

2. **Slab distribution** — Move `ClassName_alloc()`/`ClassName_free()` and `OZ_SLAB_DEFINE` into per-class files. Eliminates `oz_mem_slabs.h/.c` as a monolithic central file. Each generated `.m` → `.c` becomes self-contained.

**Pipeline for user-class source files:**
```
.m source → tree-sitter → extract_template() → Jinja template string
                                                      ↓
Clang AST → collect() → resolve() → build_context() → context dict
                                                      ↓
                                      jinja2.render(template, context) → .c
```

Foundation classes, headers, dispatch tables continue using existing `.j2` file templates.

## Part A: `extract.py`

### Step A1: `extract.py` (~200 lines)

File: `tools/oz_transpile/extract.py`

```python
def extract_template(source: bytes, tree: tree_sitter.Tree) -> str
```

**Walk algorithm** — recursive, three cases per node:
1. `node.type in OBJC_REPLACEMENTS` → call handler, emit placeholder + annotating comment
2. Leaf → source bytes verbatim
3. Interior → recurse children, copy inter-child gaps verbatim

```python
def _loc_key(node) -> str:
    line = node.start_point[0] + 1
    col = node.start_point[1] + 1
    return f"_n_{line}_{col}"
```

**Annotating comments** — the template includes comments about what the original ObjC was:

```
/* @implementation OZBlinky */

/* -[OZBlinky initWithDTSpec:] */
{{ _n_10_1 }}

/* Comment between methods preserved verbatim */

/* -[OZBlinky toggle] */
{{ _n_18_1 }}

/* @end OZBlinky */
```

For `@interface`:
```
/* @interface OZBlinky — see OZBlinky_ozh.h */
```

For `@protocol`:
```
/* @protocol Drawable — see oz_dispatch.h */
```

**`@implementation` handler** — walks children individually:
- `method_definition` → annotating comment `/* -[Class sel:] */` + `{{ _n_L_C }}` placeholder
- `comment`, `preproc_*` → verbatim from source
- `@end`, identifier, superclass_ref → structural, skip

**Method bodies** — each `method_definition` is a SINGLE placeholder. The existing `_emit_class_methods`/`_emit_compound_stmt` logic renders the full body (including ARC). Source fidelity is preserved BETWEEN methods; body internals are AST-rendered.

**Simple leaf replacements** (no placeholder, inline):
- `YES` → `true`, `NO` → `false`, `nil` → `NULL`

**Not replaced by extract.py** (inside method bodies, handled by AST emitters):
- ObjC literals, blocks, ARC retain/release, message expressions

### Step A2: Context builder (~150 lines)

New file: `tools/oz_transpile/context.py`

```python
def build_source_context(source_path: Path, module: OZModule,
                         classes: list[OZClass], stem: str,
                         root_class: str, has_item_pool: bool) -> dict[str, str]
```

For each ObjC node found by matching tree-sitter positions to OZModule data:
1. Look up OZClass/OZMethod from module
2. Call existing `_emit_*` functions → StringIO → rendered C
3. `context[_loc_key] = rendered_c_string`

String constants and block functions (hoisted) collected in `_EmitCtx` → prepended via `_preamble` key or first method's value.

### Step A3: Render + wire (~30 lines)

```python
def render_source(template_str: str, context: dict[str, str]) -> str:
    env = jinja2.Environment(undefined=jinja2.StrictUndefined)
    return env.from_string(template_str).render(**context)
```

Replace `_emit_patched_source()` internals:
```python
template_str = extract_template(source, tree)
context = build_source_context(...)
header = f'/* Auto-generated by oz_transpile -- do not edit */\n'
header += f'#include "{stem}_ozh.h"\n'
return header + render_source(template_str, context)
```

Note: `#include "oz_mem_slabs.h"` is removed from source files (see Part B).

## Part B: Slab Distribution Per-Class

### Current state (centralized)

`oz_mem_slabs.h` contains for ALL classes:
- `extern oz_slab_t oz_slab_ClassName`
- `ClassName_alloc()` / `ClassName_free()` (static inline)
- `RootClass_dispatch_free()` (switch on class_id)
- OZNumber inits, OZLock init/dealloc, OZArray/OZDictionary collection inits

`oz_mem_slabs.c`: `OZ_SLAB_DEFINE(oz_slab_ClassName, sizeof(...), count, 4)` for all classes.

### Target state (per-class)

**`ClassName_ozh.h`** (class_header.h.j2) gains:
- `extern oz_slab_t oz_slab_ClassName;`
- `static inline ClassName_alloc()` / `ClassName_free()`
- Class-specific inlines (OZNumber inits, OZLock init/dealloc, OZArray/OZDictionary collection inits)

**`ClassName_ozm.c`** (class_source.c.j2 / context builder) gains:
- `OZ_SLAB_DEFINE(oz_slab_ClassName, sizeof(struct ClassName), N, 4);`

**`oz_dispatch.h`** gains:
- `RootClass_dispatch_free()` (it already has the class enum + includes all headers)

**`oz_item_pool`** — stays shared, declared in `oz_dispatch.h`, defined in `oz_dispatch.c`

**`oz_mem_slabs.h/.c`** — ELIMINATED. Templates deleted.

### Step B1: Move alloc/free into `class_header.h.j2`

Add to end of template:
```c
extern oz_slab_t oz_slab_{{ name }};

static inline struct {{ name }} *{{ name }}_alloc(void)
{
    struct {{ name }} *obj;
    if (oz_slab_alloc(&oz_slab_{{ name }}, (void **)&obj) != 0) {
        return (struct {{ name }} *)0;
    }
    memset(obj, 0, sizeof(struct {{ name }}));
    {{ base_chain }}oz_class_id = OZ_CLASS_{{ name }};
    oz_atomic_init(&{{ base_chain }}_refcount, 1);
    return obj;
}

static inline void {{ name }}_free(struct {{ name }} *obj)
{
    oz_slab_free(&oz_slab_{{ name }}, (void *)obj);
}
```

Add class-specific inlines conditionally (OZNumber, OZLock, OZArray, OZDictionary) — move from `oz_mem_slabs.h.j2`.

### Step B2: Add `OZ_SLAB_DEFINE` to `class_source.c.j2`

```c
OZ_SLAB_DEFINE(oz_slab_{{ name }}, sizeof(struct {{ name }}), {{ pool_count }}, 4);
```

For extract.py path: the context builder emits this as a preamble in the generated `.c`.

### Step B3: Move `dispatch_free` to `oz_dispatch.h.j2`

Already includes all class headers. Add the switch function there.

### Step B4: Move `oz_item_pool` to `oz_dispatch.h/.c`

- `oz_dispatch.h`: `extern oz_mem_blocks_t oz_item_pool;`
- `oz_dispatch.c`: `OZ_MEM_BLOCKS_DEFINE(oz_item_pool, ...);`

### Step B5: Delete `oz_mem_slabs.h.j2` + `oz_mem_slabs.c.j2`

Remove templates + emission code from `emit.py`. Remove `#include "oz_mem_slabs.h"` from all templates and generated code.

### Step B6: Update golden files + tests

All golden files that reference `oz_mem_slabs.h` need updating. Tests that check for `oz_mem_slabs.h/c` files need updating.
